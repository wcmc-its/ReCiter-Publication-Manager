<objective>
Add isProxyFor check to the userfeedback save and goldstandard update API endpoints so that proxy users can save curation decisions for out-of-scope proxied persons without receiving 403 errors.

Purpose: Closes PROXY-06 -- the last remaining v1 requirement. Currently, a Curator_Scoped user assigned as a proxy for a person outside their scope can navigate to that person's curation page (client-side proxy check works), but when they try to accept/reject articles or update gold standard, the server-side scope enforcement returns 403 because the proxy check was never added to these two API files.

Output: Two modified API route files with proxy-before-scope short-circuit logic mirroring the established pattern from curate/[id].js.
</objective>
