### Component Specifications

#### ProxyBadge

```
Type: Presentational (stateless)
Library: react-bootstrap Badge (pill variant)
Props: none (static content)

Visual spec:
  - Text: "PROXY" (all caps)
  - Background: #17a2b8
  - Text color: #ffffff
  - Font size: 10px
  - Font weight: 600
  - Pill shape: border-radius fully rounded (Badge pill prop)
  - Margin: 8px left of person name
  - Vertical alignment: middle (aligned with name text baseline)
  - Min touch target: inherits from parent row (not independently clickable)

Placement:
  - Search results: inline after person name, before any action icons
  - Curation page: inline after person name in profile card area
```

#### ProxyAssignmentsSection

```
Type: Form section (controlled)
Library: react-bootstrap Form + @mui/material Autocomplete
Pattern: Mirrors CurationScopeSection exactly

Visual spec:
  - Fieldset background: #f5f5f5
  - Border: 1px solid #dee2e6
  - Border radius: 4px
  - Internal padding: 16px
  - Margin top: 8px (below CurationScopeSection)
  - Legend: "Proxy Assignments" at 20px, weight 400, line-height 1.3, color #333333
  - Helper text: "Assign specific people this user can curate beyond their scope." at 14px, weight 400, color #777777, margin-bottom 16px
  - Autocomplete: MUI Autocomplete with CssTextField (same styled component as SearchBar/CurationScopeSection)
  - Chips: MUI default chip style for selected proxies
  - Placeholder: "Search people..."

Props:
  - selectedProxies: Array of person objects
  - onProxiesChange: (values) => void
  - onSearch: (query: string) => void -- debounced at 300ms, 2-char minimum
  - searchResults: Array of person objects (max 20 results)
  - isLoading: boolean (for search loading indicator)
  - CssTextField: styled MUI TextField (passed from parent, same instance as CurationScopeSection)

Display format for options and chips: "Smith, John (jhs2001) - Medicine"

Visibility: Only rendered when user has a curation role (Curator_All, Curator_Scoped, or Curator_Self)

CSS Module: ProxyAssignmentsSection.module.css (reuse CurationScopeSection.module.css class names via shared pattern)
```

#### GrantProxyModal

```
Type: Modal (stateful)
Library: react-bootstrap Modal
Pattern: Follows Profile modal and HistoryModal patterns

Visual spec:
  - Modal size: "lg" (large)
  - Title: "Grant Proxy Access" at 20px, weight 400
  - Subtitle: "Select users who can curate publications for [Person Name]" at 14px, weight 400, color #777777
  - Body padding: 16px
  - Autocomplete: MUI Autocomplete searching admin_users with curation roles
  - Autocomplete placeholder: "Search users..."
  - Display format: "Smith, John (jhs2001)" (admin user format, no org unit)
  - Existing proxies: displayed as chips above autocomplete
  - Footer: "Discard Changes" (variant="secondary") + "Save Proxy Assignments" (variant="primary")
  - Discard Changes button: closes modal without persisting any changes
  - Save button: text "Save Proxy Assignments"
  - Close button: X in modal header (react-bootstrap default)

Props:
  - show: boolean
  - onHide: () => void
  - personIdentifier: string
  - personName: string
  - onSave: (userIds: number[]) => void

Behavior:
  - Opens from "Grant Proxy Access" button on curation page
  - Escape key closes (react-bootstrap native)
  - Focus trap active (react-bootstrap native)
  - aria-labelledby points to modal title
  - Debounce search: 300ms, 2-char minimum, max 20 results
```

#### Grant Proxy Access Button (on CurateIndividual)

```
Type: Button (triggers GrantProxyModal)
Library: react-bootstrap Button

Visual spec:
  - Variant: "outline-secondary"
  - Size: default (not sm or lg)
  - Text: "Grant Proxy Access"
  - Placement: Near person profile card, below or beside the existing action area
  - Min height: 44px (touch target requirement)

Visibility: Visible to any curator who can curate the current person
  - Curator_All: sees it for everyone
  - Curator_Scoped: sees it for in-scope + proxied people
  - Curator_Self: sees it for themselves
```

#### UsersTable Proxy Count Column

```
Location: Between existing columns (after roles, before actions)
Header text: "Proxies"
Cell content:
  - "3 people" (plural for count > 1)
  - "1 person" (singular for count = 1)
  - "--" (em dash for count = 0)
Font: Body role (14px, weight 400)
Alignment: left
Min column width: 80px
```

#### ScopeLabel Proxy Extension

```
Current format: "Curating: Faculty, Surgery"
Extended format: "Curating: Faculty, Surgery + 3 proxied people"
  - When proxy count is 0: no change to current output
  - When proxy count > 0 but no scope: "Curating: 3 proxied people"
  - When proxy count > 0 with scope: "Curating: [scope items] + N proxied people"
  - Singular: "1 proxied person"
  - Plural: "3 proxied people"
Font: Label role (12px, weight 600, color #777777)
Tooltip: full scope text + "plus N proxied people" when truncated
```

## Interaction Contracts

### ProxyAssignmentsSection Interactions

| Interaction | Behavior |
|-------------|----------|
| User types in autocomplete | Debounce 300ms, minimum 2 characters, then fetch person search results (max 20) |
| User selects a person from dropdown | Person added as chip in autocomplete |
| User clicks X on a proxy chip | Chip removed immediately (no confirmation -- matches roles/departments pattern) |
| User saves AddUser form | Proxy assignments persisted alongside other user data. Toast: "Proxy assignments saved. Changes take effect on the user's next login." |
| No search results | MUI Autocomplete shows "No people found matching your search." (noOptionsText) |
| Search in progress | MUI Autocomplete shows loading spinner (loading prop) |

### GrantProxyModal Interactions

| Interaction | Behavior |
|-------------|----------|
| "Grant Proxy Access" button clicked | Modal opens with existing proxy users pre-loaded as chips |
| User searches in modal autocomplete | Searches admin_users with curation roles, debounce 300ms, 2-char min, max 20 results |
| User selects a user from dropdown | User added as chip |
| User removes a chip | Chip removed (no confirmation) |
| "Save Proxy Assignments" clicked | POST to proxy API, close modal, show success toast |
| "Discard Changes" clicked | Modal closes, no changes persisted |
| Escape key | Modal closes, no changes persisted |
| Save fails | Modal stays open, error toast: "Unable to save proxy assignments. Please try again." |

### Search Page Proxy Interactions

| Interaction | Behavior |
|-------------|----------|
| Proxied person in search results | [PROXY] teal badge rendered inline after person name |
| "Show only people I can curate" checkbox checked | Results filtered to: (scope matches) OR (proxy assignments). Badge distinguishes which are proxy-based. |
| Click on proxied person name | Navigate to /curate/:id (no confirmation step) |
| Click curate icon on proxied person | Navigate to /curate/:id (same EditOutlined icon as scope-based results) |

### Curation Page Proxy Interactions

| Interaction | Behavior |
|-------------|----------|
| Page loads for proxied person | [PROXY] badge displayed next to person name. No banner or confirmation dialog. |
| Direct URL navigation to /curate/:id for proxied person | Access allowed -- proxy overrides scope check |
| Direct URL navigation to /curate/:id for non-proxied, non-scoped person | Redirect to /search + toast: "You do not have permission to curate publications for this person." |
| Accept/reject article for proxied person | Feedback logged with proxy user's userID in admin_feedbacklog.userID, target person's personIdentifier in admin_feedbacklog.personIdentifier |

### Loading States

| Context | Component |
|---------|-----------|
| ProxyAssignmentsSection search loading | MUI Autocomplete loading spinner (inline, within autocomplete) |
| GrantProxyModal loading existing proxies | Loader (spinner) inside modal body -- modal context per PATTERNS.md |
| GrantProxyModal search loading | MUI Autocomplete loading spinner (inline) |
| GrantProxyModal save in progress | Spinner inside "Save Proxy Assignments" button (react-bootstrap Spinner size="sm") |

### Error States

| Context | Behavior |
|---------|----------|
| Proxy search API fails | MUI Autocomplete noOptionsText: "Unable to search. Please try again." |
| Proxy save fails | Toast error: "Unable to save proxy assignments. Please try again." |
| Proxy load fails (GrantProxyModal) | Modal body shows: "Unable to load proxy data. Please close and try again." with role="alert" |
| Proxied person deleted from person table | Stale proxy -- chip still renders with personIdentifier but person name shows as "[Unknown] (jhs2001)". Harmless until proxy reassignment cleans it up. |

## Accessibility Requirements

All new components MUST meet WCAG 2.1 AA per PATTERNS.md.

| Requirement | Implementation |
|-------------|----------------|
| ProxyBadge screen reader | Badge text "PROXY" is sufficient -- screen reader reads badge content naturally |
| ProxyAssignmentsSection form labels | Form.Label "Proxy People" + Autocomplete aria-label="Proxy People" |
| GrantProxyModal focus trap | react-bootstrap Modal provides native focus trap |
| GrantProxyModal aria-labelledby | Points to modal title element |
| GrantProxyModal close | Escape key (native), close button in header, "Discard Changes" button in footer |
| "Grant Proxy Access" button | Semantic button element, min 44px touch target |
| UsersTable proxy count | Standard table cell, no special ARIA needed |
| ScopeLabel proxy text | aria-label updated to include proxy count: "Curation scope: [scope items] plus N proxied people" |
| Checkbox label | "Show only people I can curate" -- unchanged, semantically correct (proxy is included conceptually) |
