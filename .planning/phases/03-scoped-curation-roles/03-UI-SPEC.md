## Interaction Specifications

### 1. Scope Assignment (AddUser Form)

**Trigger:** User selects "Curator_Scoped" in the Roles multi-select.

**Behavior:**
- A "Curation Scope" section appears below the Roles field with a 200ms CSS transition (react-bootstrap Collapse component).
- Section has `#f5f5f5` background, 16px (md) padding, 1px solid `#dee2e6` border, 4px border-radius.
- Section heading: "Curation Scope" at 20px/400 weight.
- Helper text below heading: "Define which people this curator can manage. At least one field is required." at 14px/400 weight, `#777777` color.
- Two form groups inside:
  1. **Person Types** -- MUI Autocomplete, `multiple`, chips for selected values. Label: "Person Types". Placeholder: "Search person types..."
  2. **Organizational Units** -- MUI Autocomplete, `multiple`, chips for selected values. Label: "Organizational Units". This is the existing Departments field relocated into this section.
- When Curator_Scoped is NOT selected, the Departments field stays in its original position outside this section.

**Validation:**
- If Curator_Scoped is selected and both Person Types and Organizational Units are empty, show error text: "At least one person type or organizational unit is required" in `#dc3545` below the section.
- If Curator_All and Curator_Scoped are both selected, show error text: "Curator All and Curator Scoped cannot be combined. Remove one." in `#dc3545` below the Roles field. Disable submit button.

**Loading state:** SkeletonForm while role/department/person-type data loads.

**Success state:** Toast "User saved successfully" (existing pattern, green, top-right, 2000ms auto-close).

### 2. Scope Filter Checkbox (Search Page)

**Trigger:** User has Curator_Scoped role AND scope covers more than 1 person and fewer than all people.

**Behavior:**
- Checkbox renders in the filter area, after existing filter controls.
- Label: "Show only people I can curate" at 14px/400 weight.
- Checkbox uses react-bootstrap `<Form.Check>` with unique `controlId="scopeFilterCheckbox"`.
- When checked, search results filter to only people matching the user's scope (person type AND/OR org unit).
- Checkbox state persists across pagination (stored in Redux filters state via `updateFilters`).
- When navigating from "Curate Publications" nav link, checkbox starts pre-checked.

**Invisible when:**
- User has Curator_All role (they can curate everyone).
- User has Curator_Self only (they can only curate themselves).
- User has no curation role.

### 3. Search Results -- Scope Indicator

**In-scope people:**
- Show curate icon (EditOutlined from @mui/icons-material) in `#337ab7` at 20px, aligned right in the actions area.
- Clicking person's name navigates to `/curate/:id`.
- "Curate" action available in the actions area.

**Out-of-scope people:**
- No curate icon displayed.
- Clicking person's name navigates to `/report` with that person pre-selected as a filter.
- Only "Create Reports" action available (no "Curate" action).

**For Curator_All users:** All people show curate icon. No change from current behavior.

**For non-curator users (Reporter_All only):** No curate icon for anyone. Name click goes to report. No change from current behavior.

### 4. Scope Denial -- Curation Page

**Trigger:** Scoped curator navigates directly to `/curate/:id` for an out-of-scope person (e.g., via bookmarked URL).

**Behavior:**
- Page component checks scope on mount (after loading person data).
- If out of scope: redirect to `/search` with toast error.
- Toast: "You don't have curation access for this person" -- error theme, `#dc3545`, top-right, persistent (no auto-close -- user must dismiss).
- Console log: `[AUTH] DENY: Scoped curator {username} tried to access /curate/{id} -- not in scope`.

**API-level denial:**
- HTTP 403 response body: `{ "error": "Person not in curation scope" }`.
- No UI change needed (API consumers already handle 403).

### 5. Scope Denial -- Manage Profile Page

**Behavior:** Identical to curation page denial. Toast: "You don't have curation access for this person". Redirect to `/search`.

### 6. Navbar Scope Label

**Trigger:** User has Curator_Scoped role.

**Behavior:**
- Below the user name in the sidebar, display scope summary.
- Format: "Curating: {comma-separated list}" at 12px/600 weight.
- Text color: `#777777` (text-secondary).
- Maximum display: 3 items, then "+N more" suffix. Example: "Curating: Faculty, Staff, +2 more".
- Truncate with ellipsis if sidebar is collapsed (only visible when expanded).
- Tooltip on hover showing full list when truncated.

**For other roles:** Label does not render.

### 7. Role Filter Dropdown (Manage Users)

**Trigger:** Always visible on Manage Users page.

**Behavior:**
- react-bootstrap `<Form.Select>` above the users table, inline with search bar.
- Label: "Filter by role" (visible, with `htmlFor` association).
- Options: "All Roles" (default), "Superuser", "Curator_All", "Curator_Scoped", "Curator_Self", "Reporter_All".
- Selecting a role filters the users table to show only users with that role.
- "All Roles" clears the filter.
- Filter is additive with existing search text filter.

### 8. Scope Display in UsersTable

**Behavior:**
- Roles column currently not shown. Add a "Roles" column between "Department" and "Email" columns.
- For Curator_Scoped users: display "Curator_Scoped" followed by scope summary in parentheses.
- Format: "Curator_Scoped (Faculty, Surgery)" at 14px/400 weight.
- Scope summary inside parentheses at 12px/600 weight, `#777777` color.
- If user has multiple roles, list them comma-separated: "Curator_Scoped (Faculty), Reporter_All".
- Roles with no scope (Superuser, Curator_All, etc.) display without parentheses.

## Accessibility Requirements

All new and modified components MUST meet WCAG 2.1 AA.

| Requirement | Implementation |
|-------------|----------------|
| Scope filter checkbox | `<Form.Check>` with `controlId="scopeFilterCheckbox"` and visible label via `label` prop |
| Person Types multi-select | MUI Autocomplete with `aria-label="Person Types"` on input |
| Org Units multi-select | MUI Autocomplete with `aria-label="Organizational Units"` on input |
| Role filter dropdown | `<Form.Select>` with `<Form.Label htmlFor="roleFilter">` |
| Curate icon | `<EditOutlined titleAccess="Curate publications" />` for screen reader |
| Scope denial toast | Toast container has `role="alert"` (react-toastify default) |
| Curation Scope section | `<fieldset>` with `<legend>Curation Scope</legend>` for grouping |
| Scope label in navbar | `aria-label="Curation scope: {full list}"` on the containing element |
| UsersTable roles column | `<th scope="col">Roles</th>` header |
| Error messages | `role="alert"` on validation error text elements |
| Keyboard navigation | All new interactive elements accessible via Tab; checkboxes toggle with Space; multi-selects accessible via Enter/Arrow keys (MUI built-in) |
