<objective>
Implement client-side deduplication of PubMed search results, pending article badge with color-coded score, result count banner update, and filter state persistence across accept/reject actions.

Purpose: Curators searching PubMed from a curation page currently see articles they have already acted on, cannot distinguish pending (suggested) articles, and lose filter state when accepting/rejecting. This plan fixes all three issues in a single cohesive change.

Output: Modified TabAddPublication.tsx (dedup logic, filter fix, banner, pending-aware accept/reject), modified AddPublication.tsx (badge rendering), new CSS classes in AddPublication.module.css (badge colors).
