# Phase 17 — UI Design Contract

> Visual and interaction contract for the Admin CRUD phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | React Bootstrap 5 (tables, modals, forms, buttons) + MUI (tabs, chips, autocomplete, icons) |
| Icon library | @mui/icons-material |
| Font | Open Sans, sans-serif, Arial |

**Rationale:** This project uses a mixed Bootstrap/MUI stack with CSS Modules. No design system tool (shadcn, Tailwind) exists. All new components must follow the established dual-library pattern: Bootstrap for layout/tables/modals/forms, MUI for tabs/chips/icons/autocomplete.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline chip gaps, icon-to-text gaps |
| sm | 8px | Compact element spacing, chip internal padding |
| md | 16px | Default element spacing, cell padding, modal body sections |
| lg | 24px | Section padding within modal, toolbar row padding |
| xl | 32px | Tab panel content padding |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Existing ManageUsers page uses `15px` padding in filter bar and `5px` padding-top in places. New tab content panels use `16px` (md) for consistency but do not rewrite existing Users tab spacing.

## Layout Contract

### Tab Bar

- **Component:** MUI `<Tabs>` with `<Tab>` children
- **Variant:** `variant="standard"` (underline indicator, not contained/fullWidth)
- **Position:** Directly below the `<PageHeader label="Manage Users" />`, above all tab content
- **Tab labels:** "Users", "Roles", "Permissions" -- in that order, left-aligned
- **Active indicator:** 2px bottom border in accent color (`#0d6efd`)
- **Tab text:** 14px, weight 400 (inactive: `#666363`; active: `#0d6efd`, weight 600)
- **Tab state:** Local `useState` (integer index 0/1/2). No URL/query param changes.
- **Tab panel padding:** 16px top margin between tab bar and panel content

### Roles Tab Layout

```
[Tab Bar: Users | Roles* | Permissions]
--------------------------------------------
[ "Add Role" Button (right-aligned) ]       <- toolbar row, 16px vertical padding
--------------------------------------------
| Role Name    | Permissions          | Actions    |
|--------------|----------------------|------------|
| Superuser    | [chip][chip][chip]   | Edit | Del |
| Curator_All  | [chip][chip]         | Edit | Del |
--------------------------------------------
```

- **Table:** Bootstrap `<Table striped hover>` -- matches existing UsersTable pattern
- **Toolbar row:** Single row with "Add Role" button right-aligned. Button: Bootstrap `<Button variant="primary">`, same as existing "Add User" button style.
- **Permission chips:** MUI `<Chip size="small" label={permissionLabel} />` with default color (`#e0e0e0` background, `#333` text). Chips wrap naturally within the cell. Gap between chips: 4px (xs).
- **Action buttons:** Bootstrap `<Button variant="outline-dark" size="sm">` for "Edit". Bootstrap `<Button variant="outline-danger" size="sm">` for "Delete". 8px gap between buttons.
- **Empty state:** If no roles exist (unlikely): single table row spanning all columns with italic text "No roles defined."

### Permissions Tab Layout

```
[Tab Bar: Users | Roles | Permissions*]
--------------------------------------------
[ "Add Permission" Button (right-aligned) ] <- toolbar row
--------------------------------------------
CATEGORY: "Curation"                         <- category section header
| Key             | Label          | Category  | Resources | Actions    |
|-----------------|----------------|-----------|-----------|------------|
| canCurate       | Curate Pubs    | Curation  | 2         | Edit | Del |
| canCurateSelf   | Curate Own     | Curation  | 1         | Edit | Del |
--------------------------------------------
CATEGORY: "Administration"                   <- category section header
| Key             | Label          | Category  | Resources | Actions    |
|-----------------|----------------|-----------|-----------|------------|
| canManageUsers  | Manage Users   | Admin     | 1         | Edit | Del |
--------------------------------------------
```

- **Category grouping:** Rendered as a styled row spanning all columns before each group. Background: `#f0f0f0`, font: 14px semibold (600), color: `#333`, padding: 8px 16px. This is a visual section header row within a single `<Table>`, not separate tables.
- **Resources column:** Integer count of associated `admin_permission_resources` rows.
- **Table, toolbar, action buttons:** Same Bootstrap patterns as Roles tab.

### Role Edit Modal

```
+-------------------------------------------+
| [x] Edit Role / Add Role                  |  <- Modal.Header
+-------------------------------------------+
| Role Name: [________________]             |  <- text input
|                                           |
| Assigned Permissions:                     |  <- section label (16px semibold)
| [ ] canCurate - Curate Publications       |
| [ ] canCurateSelf - Curate Own Pubs       |
| [x] canSearch - Search Identities         |
| [x] canReport - View Reports              |
| ...                                       |
+-------------------------------------------+
| [Cancel]                    [Save Role]   |  <- Modal.Footer
+-------------------------------------------+
```

- **Modal:** React Bootstrap `<Modal>` with `show/onHide` pattern.
- **Title:** "Add Role" for create, "Edit Role" for edit. 16px semibold.
- **Role Name field:** Bootstrap `<Form.Control type="text">` full width. Required validation.
- **Permissions list:** Standard HTML checkboxes with Bootstrap `<Form.Check>` layout. Each row: checkbox + `permissionKey` in monospace + " - " + `label` in regular text. Sorted alphabetically by key.
- **Footer buttons:** "Cancel" as `<Button variant="secondary">`, "Save Role" as `<Button variant="primary">`.
- **Loading state:** Overlay Loader component centered in modal body during save.
- **Validation error:** Red text below Role Name field if empty on submit: "Role name is required." (color: `#dc3545`, font-size: 13px).

### Permission Edit Modal

```
+-------------------------------------------+
| [x] Edit Permission / Add Permission      |  <- Modal.Header
+-------------------------------------------+
| Permission Key: [can_do_thing]  (or RO)   |  <- text input (editable on create, read-only on edit)
| Label: [________________]                 |  <- text input
| Description: [________________]           |  <- text input
| Category: [Curation         v ] (combo)   |  <- Autocomplete (freeSolo)
|                                           |
| UI Resources:                             |  <- section label (16px semibold)
| +-------+------+------+------+-----+----+----+
| | Type  | Key  | Icon | Label| Route|Ord | X  |
| | [nav] | [..] | [..] | [..] | [..] |[2]| X  |
| | [tab] | [..] | [..] | [..] | [..] |[1]| X  |
| +-------+------+------+------+-----+----+----+
| [+ Add Resource]                          |  <- text button
+-------------------------------------------+
| [Cancel]              [Save Permission]   |  <- Modal.Footer
+-------------------------------------------+
```

- **Modal:** React Bootstrap `<Modal size="lg">` (large modal, resource table needs width).
- **Permission Key field:** Bootstrap `<Form.Control>` -- editable for create (D-11), rendered as `<Form.Control plaintext readOnly>` with muted background (`#e9ecef`) for edit (D-08).
- **Category field:** MUI `<Autocomplete freeSolo>` with `<TextField>` -- populated with distinct existing categories from the DB. User can select existing or type new. Matches existing Autocomplete usage in AddUser.tsx.
- **UI Resources section:** Separated by a 1px `#ddd` horizontal rule above the "UI Resources" heading.
  - Each resource: inline form row using Bootstrap grid (6 fields + remove button).
  - **resourceType:** `<Form.Select>` dropdown with options: "nav", "tab", "feature".
  - **resourceKey, icon, label, route:** `<Form.Control type="text" size="sm">`.
  - **displayOrder:** `<Form.Control type="number" size="sm">` width 60px.
  - **Remove button:** `<Button variant="link" className="text-danger">` with MUI `<ClearIcon fontSize="small" />` (already imported in TabAddPublication).
  - **"+ Add Resource" button:** Text-style button (`color: #0d6efd`, no border, underline on hover). Adds a new empty row at the bottom of the resources list.
- **Footer buttons:** Same as Role Edit Modal pattern.

## Interaction States

### Tab Switching
- **Active tab:** Underline indicator, semibold text in accent color
- **Inactive tab:** Regular weight text in secondary text color (`#666363`)
- **On switch:** Tab panel content fades in (no animation -- instant swap via conditional rendering)

### Table Rows
- **Default:** White background (odd rows), `#f5f5f5` (even rows, Bootstrap striped)
- **Hover:** Light grey highlight (Bootstrap `hover` class default)
- **No selected/active row state** -- actions are button-driven, not row-click-driven

### Buttons
- **Primary (Add/Save):** Bootstrap `btn-primary` (`#0d6efd` fill, white text). Hover: darken 10%.
- **Outline (Edit):** Bootstrap `btn-outline-dark` (`#212529` border, transparent fill). Hover: solid dark fill.
- **Danger outline (Delete in table):** Bootstrap `btn-outline-danger` (`#dc3545` border). Hover: solid red fill, white text.
- **Danger solid (Delete in confirm modal):** Bootstrap `btn-danger` (`#dc3545` fill, white text).
- **Disabled state:** When saving/loading, Save button shows `disabled` attribute with reduced opacity (0.65, Bootstrap default).

### Form Inputs
- **Default:** Bootstrap Form.Control default styling (1px `#ced4da` border, white background)
- **Focus:** Bootstrap default focus ring (`#0d6efd` with 0.25rem box-shadow)
- **Invalid:** Red border (`#dc3545`), red error text below field (13px)
- **Read-only (permissionKey on edit):** Background `#e9ecef`, no focus ring, cursor default

### Loading States
- **Tab content loading:** Centered `<Loader />` component (existing spinner) replacing table content
- **Modal save in progress:** "Save" button disabled + Loader overlay inside modal body
- **Toast on complete:** `toast.success("Role created successfully")` / `toast.error("Failed to save role")` at position `top-right`, `autoClose: 2000`, `theme: 'colored'`

### Chip Interactions (Roles Tab)
- **Permission chips in role rows:** Non-interactive (display only). MUI `<Chip size="small" />` with no `onClick` or `onDelete`. Default style (`#e0e0e0` background).
- **No chip removal from the table view** -- editing happens in the modal.

## Accessibility Notes

- **Tab navigation:** MUI Tabs provides built-in keyboard navigation (arrow keys between tabs, Enter/Space to select). Ensure `aria-controls` and `id` attributes connect each Tab to its TabPanel.
- **Modals:** React Bootstrap Modal provides focus trapping and `aria-labelledby` by default. Ensure modal title has an `id` that matches `aria-labelledby` on the Modal.
- **Tables:** Use `<th scope="col">` on header cells. Empty state messages use `<td colSpan={N}>` to span the full table width.
- **Form validation:** Error messages linked to inputs via `aria-describedby`. Invalid inputs have `aria-invalid="true"`.
- **Delete buttons:** Include `aria-label="Delete [ItemName]"` for screen reader context since the button text is just "Delete".
- **Chips:** Non-interactive chips do not need additional ARIA -- MUI Chip renders as `<div role="button">` only when clickable; for display-only it renders as a plain `<div>`.
