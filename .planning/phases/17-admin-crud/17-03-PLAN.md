**Component `PermissionsTab`** (FunctionComponent, default export):

State variables:

- `loading: boolean`, initially `true`
- `showEditModal: boolean`, initially `false`

**`fetchPermissions()` function:**
- Call `fetch('/api/db/admin/permissions', { credentials: "same-origin", method: 'GET', headers: { Accept: 'application/json', "Content-Type": "application/json", 'Authorization': reciterConfig.backendApiKey } })`
- Parse response JSON, set `permissions` state. Set `loading` to `false`.
- On error: `toast.error("Failed to load permissions. Please try again.", { position: "top-right", autoClose: 2000, theme: "colored" })`.

**`useEffect`:** Call `fetchPermissions()` on mount.

**`handleDeleteClick(permission)` function:**
- Same two-phase pattern as RolesTab: call DELETE with `?check=true`.
- If 409: set `deleteBlockedInfo` to `{ name: permission.label, roles: data.roles }`, show `DeleteBlockedModal` with `itemType="permission"`.
- If 200 with `canDelete: true`: set `deletingPermission`, show `DeleteConfirmModal`.

**`handleDeleteConfirm()` function:**
- Call DELETE without `?check=true` on `deletingPermission.permissionID`.
- On success: toast success `"[label] deleted successfully"`, `fetchPermissions()`.
- On error: toast error.
- Close modal.

**`handleSavePermission(permissionData)` callback (passed to PermissionEditModal):**
- If `editingPermission` is not null (edit mode): PUT to `/api/db/admin/permissions/${editingPermission.permissionID}` with body `{ label, description, category, resources }`.
- If `editingPermission` is null (create mode): POST to `/api/db/admin/permissions/create` with body `{ permissionKey, label, description, category, resources }`.
- On success (201 or 200): close modal, toast success, `fetchPermissions()`.
- On 409 (duplicate key): `toast.error("A permission with this key already exists.", ...)`.
- On other error: toast error.

**Category grouping render logic per UI-SPEC and RESEARCH.md Pitfall 7:**

The permissions are already sorted by `[category ASC, permissionKey ASC]` from the server. Render the table by iterating through permissions and inserting a category header row whenever the category changes:

```tsx
{loading ? <Loader /> : (
  <Table striped hover>
    <thead>
      <tr>
        <th scope="col">Key</th>
        <th scope="col">Label</th>
        <th scope="col">Category</th>
        <th scope="col">Resources</th>
        <th scope="col">Actions</th>
      </tr>
    </thead>
    <tbody>
      {permissions.length === 0 ? (
        <tr><td colSpan={5}><em>No permissions defined.</em></td></tr>
      ) : (() => {
        let lastCategory = '';
        const rows = [];
        permissions.forEach((perm, idx) => {
          if (perm.category !== lastCategory) {
            lastCategory = perm.category;
            rows.push(
              <tr key={`cat-${perm.category}`} className={styles.categoryHeader}>
                <td colSpan={5}>{perm.category}</td>
              </tr>
            );
          }
          rows.push(
            <tr key={perm.permissionID}>
              <td><code style={{ fontSize: '13px' }}>{perm.permissionKey}</code></td>
              <td>{perm.label}</td>
              <td>{perm.category}</td>
              <td>{perm.resourceCount}</td>
              <td>
                <Button variant="outline-dark" size="sm" onClick={() => { setEditingPermission(perm); setShowEditModal(true); }} style={{ marginRight: 8 }}>Edit</Button>
                <Button variant="outline-danger" size="sm" onClick={() => handleDeleteClick(perm)} aria-label={`Delete ${perm.label}`}>Delete</Button>
              </td>
            </tr>
          );
        });
        return rows;
      })()}
    </tbody>
  </Table>
)}
```
