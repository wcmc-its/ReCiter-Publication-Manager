# Phase 2 — UI Design Contract

> Visual and interaction contract for the Group Curation view redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker. This contract also documents the current design baseline for the full audit (PATTERNS.md will extend this).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | react-bootstrap 2.x + @mui/material (hybrid -- Bootstrap primary, MUI for icons + list components) |
| Icon library | @mui/icons-material |
| Font | Open Sans, sans-serif, Arial (headings); Helvetica Neue, Helvetica, sans-serif (body -- Bootstrap default) |

**Note:** No design system initialization (no shadcn, no Tailwind). The project uses Bootstrap 5 via react-bootstrap with CSS Modules for component-scoped styles and Bootstrap utility classes for layout. This UI-SPEC prescribes target values; PATTERNS.md will document current inconsistencies and migration path.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, inline button margins |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, accordion body padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical spacing (above/below content areas) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: 44px minimum height/width for all interactive elements (WCAG 2.5.5 Target Size)
- Sidebar collapsed width: 82px (existing, preserved)
- Sidebar expanded width: 250px (existing, preserved)
- Header height: 60px (existing, preserved)
- Main content padding: 20px vertical, 40px horizontal (existing AppLayout.module.css, mapped to lg/xl)

**Current state:** Spacing is ad-hoc (5px, 10px, 15px, 20px, 23px, 25px, 30px, 40px, 50px used across components). The audit will flag deviations. Group Curation redesign must use only the values above.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Article metadata, evidence text, filter labels |
| Label | 12px | 600 (semibold) | 1.4 | Badges, status tags, button text, small captions |
| Heading | 20px | 400 (regular) | 1.3 | Page titles (h1), section headers |
| Display | 28px | 500 (medium) | 1.2 | Score display, person count summary |

**Font stack:** `"Open Sans", sans-serif, Arial` for all text. Bootstrap default (`system-ui, -apple-system, "Segoe UI", Roboto, ...`) as fallback.

**Current state observations:**
- h1 is globally styled at 20px weight 200 (`globals.css` line 87-92) -- audit will evaluate whether weight 200 is too light
- CuratePublications sectionHeader uses 28px weight 500 (`CuratePublications.module.css` line 10)
- Publication evidence table uses 14px (`Publication.module.css` line 32)
- Publication buttons use 12px weight 600 (`Publication.module.css` line 67-76)
- Font weights used across codebase: 200, 300, 400, 500, 600, 700, 800 -- audit will flag and recommend consolidating to 400 + 600

## Copywriting Contract

### Group Curation View (Redesign Target)

| Element | Copy |
|---------|------|
| Primary CTA | "View All Pending" (navigates to individual curation for that person) |
| Secondary CTA | "Accept" / "Reject" (per-article action buttons, existing pattern) |
| Page heading | "Curate Publications" |
| Summary line | "{count} people with pending publications" (existing: "About {count} people with pending publications") |
| Empty state heading | "No pending publications" |
| Empty state body | "All publications for the selected group have been curated. Return to Find People to select a different group." |
| Empty state (no selection) | "Select people from Find People to begin group curation." |
| Error state (API failure) | "Unable to load publications. The scoring service may be temporarily unavailable. Try refreshing the page." |
| Error state (no Redux data) | "No curators selected. Go to Find People to select people for group curation." |
| Pagination | "Previous" / "Next" (existing, with MUI NavigateBefore/NavigateNext icons) |
| Filter bar | "Update Search..." button label (existing, navigates back to /search) |
| Person card header | "{Full Name}" with title subtitle |
| Articles remaining | "{count} Pending" badge on person card |
| View profile action | "View Profile" (opens modal) |
| Undo action | "Undo" (reverts accept/reject) |
| Evidence toggle | "Show Evidence" / "Hide Evidence" (per-article expandable section) |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Reject article | No confirmation -- immediate action with "Undo" available (existing pattern, no change) |
| Accept article | No confirmation -- immediate action with "Undo" available (existing pattern, no change) |

**Rationale:** Accept/Reject are high-frequency curation actions. Confirmation dialogs would create friction that contradicts the "blitz through easy accepts" design goal (CONTEXT.md). Undo availability is the safety net.

### Interaction Patterns

| Pattern | Implementation |
|---------|---------------|
| Loading | SkeletonCard (3 placeholders) while `publicationsGroupDataFetching` is true |
| Person expand/collapse | Bootstrap Accordion (existing pattern). First person expanded by default. |
| Accept/Reject feedback | Button changes to "Undo" state. Article fades out (opacity transition 300ms). Count decrements. |
| Pagination | Previous/Next buttons. Server-side pagination via Redux (existing `publicationsFetchGroupData` with slice). |
| Sort | Client-side sort of loaded person list. Default: highest scoring article first (CONTEXT.md decision). |
| Navigation to individual | Click person name or "View All Pending" navigates to `/curate/{personIdentifier}` |
| View Profile | Opens Profile modal overlay (existing pattern) |
| Error recovery | "Try again" button re-dispatches `publicationsFetchGroupData` |
| No Redux data (direct nav to /curate) | Show "no selection" empty state with link to /search |

```
+----------------------------------------------------------+
| Header (sticky, #cf4520, 60px)                           |

| Nav    |                                                 |

|        |                                                 |
|        |  [Summary: "X people..." | Sort | Prev | Next] |
|        |                                                 |
|        |  [Person Card 1 - Expanded]                     |
|        |    Name + Title        [View Profile] [View All]|
|        |    Article 1: citation [Accept] [Reject]        |
|        |    Article 2: citation [Accept] [Reject]        |
|        |    ... (max 4 per person)                        |
|        |    [View All X Pending]                          |
|        |                                                 |
|        |  [Person Card 2 - Collapsed]                    |
|        |    Name + Title + Pending Badge                 |
|        |                                                 |
|        |  [Person Card 3 - Collapsed]                    |

|        |                                                 |
+--------+-------------------------------------------------+
| Footer                                                   |
+----------------------------------------------------------+
```

**Key layout decisions:**
- Person cards stack vertically, full width of main content area
- No side-by-side card layout -- articles within each person need horizontal space for citation text + action buttons
- Filter bar sits below page title, above summary line (existing pattern, preserved)
- Pagination appears both above and below the person list (existing: top only; recommend adding bottom for long lists)
- Max 4 articles shown per person in group view (existing `maxArticlesPerPerson` config); "View All" links to individual curation

## Accessibility Contract

| Requirement | Target | Verification |
|-------------|--------|-------------|
| WCAG compliance | 2.1 AA | Lighthouse accessibility score 90+ |
| Keyboard navigation | Full tab-order through filter bar, sort, person cards, article actions | Manual testing per view |
| Focus indicators | Visible focus ring on all interactive elements (Bootstrap default `:focus-visible`) | Manual + Lighthouse |
| Screen reader | Person cards announced with name + pending count; articles with title + score | aria-label on containers |
| Color contrast | 4.5:1 for normal text, 3:1 for large text | Lighthouse automated check |
| Accept/Reject buttons | Minimum 44x44px touch target | Enforced via `min-height: 44px; min-width: 44px` |
| onClick on non-button elements | Must have `role="button"`, `tabIndex={0}`, `onKeyDown` handler | eslint-plugin-jsx-a11y strict |
