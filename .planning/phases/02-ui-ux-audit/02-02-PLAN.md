  <action>
Synthesize findings from all 8 audit reports into a prescriptive design patterns document. This document will be the authoritative reference for all future UI work in Phase 3+.

PATTERNS.md must be PRESCRIPTIVE ("do this") not just DESCRIPTIVE ("this is what exists"). For each pattern area, document: what the current state is, what the target state is, and what the migration path is.

**Required sections (4 focus areas from CONTEXT.md):**

**1. Component Reuse Patterns**
- Which existing components should be reused as-is (list with file paths)
- Which existing components need refactoring before reuse (with specific issues)
- Which new components should be created (gaps identified during audit)
- Naming conventions: PascalCase component files, `{ComponentName}.module.css` for styles
- When to use react-bootstrap vs @mui/material vs custom: prescribe a clear rule
  - Recommendation: react-bootstrap for layout/form/interactive components (Button, Modal, Form, Accordion, Table, Pagination)
  - @mui/material for icons only (@mui/icons-material)
  - Custom CSS Modules for component-specific styling
  - Bootstrap utility classes (d-flex, justify-content-center, etc.) for quick layout
  - Do NOT mix: a component should use react-bootstrap OR custom, not both for the same purpose

**2. Spacing and Layout System**
- Document the target spacing scale from 02-UI-SPEC.md: xs(4px), sm(8px), md(16px), lg(24px), xl(32px), 2xl(48px)
- Document current inconsistencies found across all 8 audits (list specific deviations with file:line)
- Prescribe: new/modified components MUST use only these spacing values
- Prescribe: use Bootstrap spacing utilities where possible (m-1=4px through m-5=48px mapping)
- Document the AppLayout structure: sidebar (82/250px), header (60px), main content padding (20px vertical, 40px horizontal)
- Touch target minimum: 44px for all interactive elements

**3. Color and Typography**
- Document the full color palette from 02-UI-SPEC.md with role assignments
- Document current color inconsistencies found across audits
- Prescribe: accent color (#cf4420 WCM orange) is header-only, never for buttons/links
- Prescribe: primary interactive color is #337ab7 for links and text buttons
- Document allowed font weights: 400 (regular) and 600 (semibold) ONLY
- Document current weight violations found across audits (weights 200, 300, 500, 700, 800 must be eliminated)
- Prescribe: typography roles (Body 14px/400, Label 12px/600, Heading 20px/400, Display 28px/600)
- Font stack: "Open Sans", sans-serif, Arial

**4. Interaction Patterns**
- Loading states: prescribe SkeletonTable/SkeletonCard/SkeletonProfile/SkeletonForm for page-level loading, Loader.tsx for modal contexts only
- Error states: prescribe pattern (descriptive message + actionable guidance + retry button where applicable)
- Empty states: prescribe pattern (clear heading + explanation + navigation CTA)
- Modals: prescribe Bootstrap Modal pattern, accessible (focus trap, aria-labelledby, Escape to close)
- Toasts: prescribe react-toastify pattern (top-right, 2000ms autoClose for success, persistent for errors)
- Dropdowns/selects: prescribe Bootstrap Form.Select with proper label
- Accordion: prescribe Bootstrap Accordion with keyboard support
- Pagination: prescribe shared Pagination component pattern
- Accept/Reject: prescribe immediate action with Undo (no confirmation dialog)

**Additional sections:**

**5. Accessibility Standards**
- WCAG 2.1 AA compliance target
- eslint-plugin-jsx-a11y strict mode enforced
- All onClick on non-button elements must have role="button", tabIndex={0}, onKeyDown handler -- OR use a semantic button element (preferred)
- All form inputs must have associated labels (htmlFor or aria-label)
- All images must have alt text
- Color contrast: 4.5:1 for normal text, 3:1 for large text
- Focus indicators: use Bootstrap default :focus-visible (do not override with outline: none)

**6. Cross-View Issue Summary**
- Aggregate finding counts by severity across all 8 views
- Top 5 most common issues (with counts)
- Priority fix list for Phase 2 (Critical items only)
- Deferred fix list for Phase 3+ (with effort estimates)
