**IMPORTANT:** Read ALL 8 AUDIT-*.md "Fix Now (Phase 2)" sections first to build the complete fix list. Then apply fixes systematically. The exact fixes depend on audit findings -- the categories below are the expected fix types based on pre-audit research.

**Fix Strategy:**
- For fragile components (Publication.tsx 690 lines, TabAddPublication.tsx 629 lines): extract the section being fixed into a sub-component, then apply the a11y fix to the extracted component. This isolates changes and reduces regression risk.
- For non-fragile components: apply surgical fixes directly.
- For each fix: reference the specific audit finding (AUDIT file, finding #) in a code comment.

**Expected fix categories (confirm against actual audit findings):**

**Category 1: onClick on non-interactive elements**
For each `onClick` on a `div`, `span`, `td`, or `tr`:
- PREFERRED: Replace the non-interactive element with a semantic `<button>` element with appropriate styling (`btn btn-link`, `border-0 bg-transparent`, etc.)
- ALTERNATIVE (when button replacement would break layout): Add `role="button"`, `tabIndex={0}`, and `onKeyDown` handler:
  ```tsx
  <div
    onClick={handler}
    onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); handler(); } }}
    role="button"
    tabIndex={0}
  >
  ```

**Category 2: Missing img alt text**
- Identity.js line 33 (profile photo): Add `alt={`Profile photo of ${firstName} ${lastName}`}`
- Any other images found during audit: add descriptive alt text or `alt=""` for decorative images

**Category 3: Missing form labels**
- For each input without a label:
  - Add `<label htmlFor="input-id" className="visually-hidden">Label Text</label>` if label should be visually hidden
  - Or add `aria-label="Label Text"` directly on the input
  - Ensure every `<input>`, `<select>`, and `<textarea>` has an associated label

**Category 4: autoFocus removal**
- Login.js: Remove `autoFocus` attribute from the login input (jsx-a11y/no-autofocus violation in strict mode)

**Category 5: Other Critical fixes from audits**
- Apply any additional Critical-severity fixes identified in the audit "Fix Now" sections
- Each fix should be the minimal change needed -- do not refactor beyond what's required for the fix
