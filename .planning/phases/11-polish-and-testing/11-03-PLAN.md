**Primary fix pattern -- replace non-interactive element with `<button>`:**
Per the UI-SPEC visual contract, when replacing `<span onClick>` or `<div onClick>` with `<button>`:
```tsx
// BEFORE
<span onClick={handler} style={{ cursor: 'pointer' }}>Click me</span>

// AFTER
<button
  type="button"
  onClick={handler}
  style={{
    background: 'none',
    border: 'none',
    padding: 0,
    font: 'inherit',
    color: 'inherit',
    cursor: 'pointer',
  }}
>
  Click me
</button>
```

The button MUST be visually identical to the original element. The inline styles `background: none; border: none; padding: 0; font: inherit; color: inherit;` achieve this. If the original element had additional styles, preserve them.

**For `<a onClick>` without href (anchor-is-valid in Header.tsx line 22):**
```tsx
// BEFORE
<a onClick={handler}>Logout</a>

// AFTER
<button
  type="button"
  onClick={handler}
  style={{
    color: 'var(--wcm-red)',
    textDecoration: 'none',
    cursor: 'pointer',
    background: 'none',
    border: 'none',
    padding: 0,
    font: 'inherit',
  }}
>
  Logout
</button>
```

**For `<div onClick>` with display: block semantics (if any):**
Add `display: block; width: 100%;` to the button style.

**For elements that cannot be changed to `<button>` (e.g., `<tr>`, `<li>` with click handlers -- check no-noninteractive-element-interactions, 2 instances):**
```tsx
<tr
  onClick={handler}
  role="button"
  tabIndex={0}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handler(e);
    }
  }}
>
```
