3. Fix all `label-has-associated-control` violations (22 errors). The fix pattern for each:

   **Pattern A -- label with nearby input that has an id:**
   Add `htmlFor="matching-id"` to the `<label>` element, where `matching-id` matches the `id` attribute on the associated `<input>`, `<select>`, `<Autocomplete>`, or `<TextField>`.
   
   Example (AddUser.tsx):
   ```tsx
   // BEFORE
   <label className={styles.fieldLabel}>First Name</label>
   <input id="firstName" ... />
   
   // AFTER
   <label className={styles.fieldLabel} htmlFor="firstName">First Name</label>
   <input id="firstName" ... />
   ```

   **Pattern B -- label wrapping an input (no id needed):**
   If the label directly wraps the input element, the association is implicit and jsx-a11y should recognize it. If it does not (false positive from React Bootstrap Form.Check or MUI components), add an eslint-disable comment:
   ```tsx
   {/* eslint-disable-next-line jsx-a11y/label-has-associated-control -- Form.Check handles association internally */}
   ```

   **Pattern C -- MUI Autocomplete with label:**
   The CurationScopeSection and ProxyAssignmentsSection have `<label>` elements above `<Autocomplete>` components. The Autocomplete renders an `<input>` with an auto-generated id. Fix: add `htmlFor` matching the Autocomplete's `id` prop.
   
   CurationScopeSection.tsx line ~34:
   ```tsx
   <label className={styles.fieldLabel} htmlFor="scope-person-types">Person type(s)</label>
   ```
   Line ~60:
   ```tsx
   <label className={styles.fieldLabel} htmlFor="scope-org-units">Organizational unit(s)</label>
   ```
   
   ProxyAssignmentsSection.tsx line ~74:
   ```tsx
   <label className={styles.fieldLabel} htmlFor="proxy-assignments">People this user can curate on behalf of</label>
   ```

   **Specific file fixes:**
   
   - **AddUser.tsx (10 violations):** Read the file to identify each `<label>` without `htmlFor`. For each, find the corresponding input/select element and add `htmlFor` matching its `id`. If the input has no `id`, add one (use descriptive ids like `adduser-firstName`, `adduser-lastName`, `adduser-email`, `adduser-status`, `adduser-personIdentifier`, etc.).
   
   - **AdminSettings.tsx (3 violations):** Similar label/input pattern. Add `htmlFor` to match each input's `id`. If inputs lack ids, add them.
   
   - **Notifications.tsx (2 violations at lines 174, 184):** These are `<label>` elements for notification frequency/threshold inputs. Add `htmlFor` matching the input `id`.
   
   - **TabAddPublication.tsx (2 violations at lines 452, 466):** Add `htmlFor` to labels, add `id` to associated inputs if missing.
   
   - **DatePicker.tsx (2 violations at lines 78, 90):** Labels for date picker inputs. Add `htmlFor` matching the date picker's rendered input `id`. If using react-datepicker, it accepts an `id` prop.
   
   - **SearchSummary.tsx (1 violation at line 420):** Single label fix.
